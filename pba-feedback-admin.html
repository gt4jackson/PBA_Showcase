📐 Adding a second scenario to an existing task+construct slot

The scenario map below is structured as arrays — each slot can hold multiple scenario files. When a slot has more than one, the hub card should offer a version selector. Here's how to extend:

1
Create the new scenario JSON — e.g. scenarios/the-brief-project-management-v2.json with adjusted weightings or a different case study. Follow the existing schema for that task type.
2
Add it to the SCENARIO_MAP in this file — append to the array for that task+construct pair. The coverage matrix will automatically show both chips.
3
Update TASK_CONSTRUCT_MAP in index.html — change the single scenario: '...' value to an array, and add a version-selector UI to the card (or use URL params to let the user pick at launch).

The simplest hub UI for version selection: a small v1 · v2 toggle appended to the card footer, where each version links to the shell with ?scenario=.... No changes to the task shell itself are needed.

// index.html — multi-scenario slot example
'project-management': {
  'the-brief': [
    { scenario: 'scenarios/the-brief-project-management.json',    label: 'v1 — Project Recovery',  dims: [...] },
    { scenario: 'scenarios/the-brief-project-management-v2.json', label: 'v2 — Scope Trade-offs',  dims: [...] }
  ]
}
